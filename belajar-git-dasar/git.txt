------------------------------
Git adalah sebuah alat atau sistem yang digunakan untuk mengatur dan melacak setiap perubahan pada file proyek, terutama pada kode program.
Dengan menggunakan Git, kita dapat mencatat setiap perubahan yang terjadi, sehingga jika terjadi kesalahan atau ingin melihat versi sebelumnya, kita bisa mengembalikannya dengan mudah.

Git membuat pekerjaan pengembangan perangkat lunak menjadi lebih teratur, aman, dan efisien, baik saat dikerjakan sendiri maupun dalam tim.


ğŸ•“ SEBELUM ADANYA GIT
------------------------------
Sebelum Git dibuat, para pengembang telah menggunakan sistem Version Control System (VCS) untuk mengatur versi kode.
Namun, sistem lama memiliki banyak keterbatasan.

1. CVS (Concurrent Versions System) dan Subversion (SVN)
   - Menggunakan sistem terpusat (centralized), semua data proyek tersimpan di satu server utama.
   - Kekurangan:
     â€¢ Jika server utama rusak, seluruh data bisa hilang.
     â€¢ Developer harus selalu online untuk bekerja.

2. BitKeeper
   - Digunakan oleh pengembang Kernel Linux karena lebih canggih dari CVS dan SVN.
   - Namun bersifat komersial (berbayar) dan bukan open-source.
   - Karena perbedaan lisensi, proyek Linux tidak dapat lagi menggunakan BitKeeper.

Akibat hal itu, Linus Torvalds dan komunitas Linux membuat alat baru yang:
   - Gratis dan open-source
   - Cepat dan efisien
   - Aman serta dapat digunakan offline

ğŸ§  CARA KERJA GIT
------------------------------
Git bekerja dengan menyimpan snapshot (salinan keadaan) dari proyek setiap kali melakukan commit.
Namun Git tidak menyimpan seluruh file berulang kali, hanya menyimpan perbedaan (delta) dari versi sebelumnya.

Struktur kerja Git terdiri dari:

1. Repository (Repo)
   - Menyimpan semua data proyek dan riwayat perubahan.
   - Jenis:
     â€¢ Local Repository â†’ tersimpan di komputer.
     â€¢ Remote Repository â†’ tersimpan di server (misalnya GitHub).

2. Working Directory
   - Tempat kita bekerja dan mengedit file proyek.

3. Staging Area
   - Tempat sementara sebelum perubahan disimpan ke repository.

4. Commit
   - Proses menyimpan perubahan dari staging area ke repository disertai pesan (message).

5. Branch dan Merge
   - Branch digunakan untuk membuat cabang pengembangan baru.
   - Merge digunakan untuk menggabungkan cabang ke main branch.

6. Push dan Pull
   - git push â†’ mengirim perubahan dari local ke remote repository.
   - git pull â†’ mengambil perubahan dari remote ke local repository.


âš™ï¸ KONFIGURASI AWAL GIT
------------------------------
Sebelum menggunakan Git, lakukan konfigurasi identitas agar setiap commit tercatat dengan benar.

Perintah konfigurasi:

   git config --global user.name "Nama Kamu"
   git config --global user.email "emailkamu@example.com"

Keterangan:
   - Opsi --global membuat konfigurasi berlaku untuk semua repository di komputer.
   - Untuk konfigurasi hanya pada satu proyek, hilangkan opsi --global.

Cek hasil konfigurasi:
   git config --list

ğŸš€ CONTOH IMPLEMENTASI GIT DARI AWAL
------------------------------
1. Buat folder proyek baru:
      mkdir latihan-git
      cd latihan-git

2. Inisialisasi repository:
      git init

3. Konfigurasi user (jika belum):
      git config --global user.name "Nama Kamu"
      git config --global user.email "emailkamu@example.com"

4. Tambahkan file dan lakukan commit:
      echo "Hello Git" > README.md
      git add .
      git commit -m "Commit pertama"

5. Hubungkan ke repository GitHub:
      git remote add origin https://github.com/username/latihan-git.git

6. Kirim perubahan ke GitHub:
      git push -u origin main

7. Jika ada perubahan di GitHub:
      git pull origin main
